📌 વ્યાખ્યા (Definition)

🔢 3. Basic Arithmetic in Encryption (Encryption ની ગાણિતિક ક્રિયાઓ) ★★★★★ ▼

📌 Modular Arithmetic ★★★★★

Modular Arithmetic = Encryption ની ગણિત ની base. Numbers ને specific range (0 to n-1) માં wrap around.

a mod n = r (જ્યાં 0 ≤ r < n) ઉદાહરણ: 17 mod 5 = 2 (કારણ કે 17 = 3×5 + 2)
Clock Arithmetic: 3 + 11 = 14 → 14 mod 12 = 2 (Clock ઉપર 2 વાગ્યા!)
OperationExampleResult
Addition mod(7 + 8) mod 123
Subtraction mod(5 − 9) mod 73
Multiplication mod(4 × 5) mod 76
Power mod3² mod 54

📌 Prime Numbers & GCD ★★★★

Prime Number: ફક્ત 1 અને પોતે જ divisible (2, 3, 5, 7, 11, 13...)

GCD (Greatest Common Divisor): Euclidean Algorithm

GCD(48, 18): 48 = 2 × 18 + 12 18 = 1 × 12 + 6 12 = 2 × 6 + 0 ∴ GCD = 6
Coprime Numbers: GCD(a, b) = 1 → RSA encryption માં ઉપયોગ!

📌 Euler's Totient Function (φ) ★★★★

φ(n) = 1 to n range ના numbers જે n ના coprime છે For prime p: φ(p) = p - 1 For p,q prime: φ(p×q) = (p-1)(q-1) ઉદાહરણ: φ(7) = 6 (1,2,3,4,5,6 – all coprime with 7) φ(15) = φ(3×5) = (3-1)(5-1) = 2×4 = 8

📌 XOR Operation ★★★★★

XOR (Exclusive OR) = Encryption ની સૌથી મહત્વની operation

Rules: 0 XOR 0 = 0 0 XOR 1 = 1 1 XOR 0 = 1 1 XOR 1 = 0 Property: A XOR A = 0 (Same = Cancel) A XOR 0 = A (Zero = No change) (A XOR B) XOR B = A (Reversible!)
Encryption: Plaintext XOR Key = Ciphertext
Decryption: Ciphertext XOR Key = Plaintext
ઉદાહરણ: 1010 XOR 1100 = 0110

📌 Number Systems ★★★

SystemBaseDigitsઉદાહરણ
Binary20,11010
Octal80-717
Decimal100-910
Hexadecimal160-9, A-F1A3F
Binary → Decimal: 1011 = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8+0+2+1 = 11 Hex → Decimal: FF = 15×16 + 15 = 255

📌 Caesar Cipher (Classic Encryption) ★★★★

સૌથી જૂૂની encryption. દરેક letter ને fixed number (key) positions shift.

Encryption: C = (P + K) mod 26 Decryption: P = (C - K) mod 26 ઉદાહરણ (K=3): A(0) → D(3), B(1) → E(4), Z(25) → C(2) "HELLO" → "KHOOR"
Julius Caesar K=3 ઉપયોગ કરતા. Brute Force attack = 25 tries only → Weak!

📌 Hash Functions ★★★★★

Hash Function: Any input → Fixed size output (digest). One-way function – reverse impossible.

AlgorithmOutput SizeStatusઉપયોગ
MD5128 bitsWeak (broken)File verification
SHA-1160 bitsDeprecatedOld SSL
SHA-256256 bitsStrong ✓Bitcoin, TLS
SHA-512512 bitsStrongest ✓High security
Properties: 1. Deterministic (same input → same output always) 2. Fast computation 3. Avalanche Effect (small change → completely different hash) 4. Pre-image resistance (hash → input impossible) 5. Collision resistance (two inputs → same hash extremely rare)

🔑 4. Symmetric Encryption Algorithms (સમ-key Encryption) ★★★★★ ▼

📌 Symmetric Encryption શું છે? ★★★★★

Symmetric Encryption = Encryption અને Decryption બંને માટે એક જ key ઉપયોગ. "Secret Key Cryptography" પણ કહેવાય.

Encryption: Plaintext + Key → Ciphertext Decryption: Ciphertext + Key → Plaintext (Same Key Used Both Ways)
ઉદાહરણ: Lock-Key system – ताळो ખોલવા-બંધ કરવા same key

🔐 DES (Data Encryption Standard) ★★★★★

IBM 1970s developed, NIST 1977 standard. Now considered WEAK.

DES Details: • Key: 56-bit effective (64-bit with parity bits) • Block: 64-bit • Rounds: 16 Feistel rounds • Structure: Feistel Network • Developed: IBM, 1977 NIST Standard • Broken: 1999 (DES Cracker)
DES weakness: 56-bit key = 2⁵⁶ ≈ 72 quadrillion combinations – Modern computers crack in hours!

DES Process (Feistel Structure):

  1. Initial Permutation (IP) – 64-bit input
  2. 16 Rounds of Feistel function
  3. Each round: expand, XOR with subkey, S-box substitution, permutation
  4. Final Permutation (IP⁻¹)

🔐 AES (Advanced Encryption Standard) ★★★★★

DES ને replace કરવા NIST 2001 standard. Rijndael algorithm based. Most widely used.

AES Details: • Block Size: Always 128-bit • Key Sizes: 128, 192, 256 bits • Rounds: 10 (128), 12 (192), 14 (256) • Operations: SubBytes, ShiftRows, MixColumns, AddRoundKey • Structure: Substitution-Permutation Network (SPN)

AES 4 Main Operations:

StepOperationPurpose
1SubBytesS-Box substitution (non-linearity)
2ShiftRowsRow-wise shift (diffusion)
3MixColumnsColumn mixing (diffusion)
4AddRoundKeyXOR with round key (confusion)
AES ઉપયોગ: Wi-Fi (WPA2/WPA3), HTTPS/TLS, VPN, Disk Encryption (BitLocker), WhatsApp

🗝️ 5. Asymmetric Key Encryption (અ-સમ key Encryption) ★★★★★ ▼

📌 Asymmetric Encryption શું છે? ★★★★★

Asymmetric Encryption = બે અલગ keys ઉપયોગ: Public Key (encrypt) + Private Key (decrypt). "Public Key Cryptography" પણ કહેવાય.

Encryption: Plaintext + Public Key → Ciphertext Decryption: Ciphertext + Private Key → Plaintext (Different Keys for Encrypt & Decrypt)
ઉદાહરણ: Mailbox – સૌ letter નાખી શકે (Public Key = mail slot), ફક્ત owner ખોલી શકે (Private Key)

🔐 RSA Algorithm ★★★★★

RSA = Rivest-Shamir-Adleman (1977). Most widely used asymmetric algorithm. Based on difficulty of factoring large numbers.

RSA Key Generation Steps: 1. Choose two large primes: p, q 2. Calculate: n = p × q 3. Calculate: φ(n) = (p-1)(q-1) 4. Choose e: 1 < e < φ(n), gcd(e, φ(n)) = 1 5. Find d: d × e ≡ 1 mod φ(n) (Multiplicative Inverse) Public Key = (e, n) Private Key = (d, n) Encryption: C = Mᵉ mod n Decryption: M = Cᵈ mod n

RSA Example (Simple):

p = 3, q = 11 n = 3 × 11 = 33 φ(n) = (3-1)(11-1) = 20 Choose e = 3 (gcd(3,20) = 1) ✓ d = 7 (because 3×7 = 21 ≡ 1 mod 20) ✓ Public Key = (3, 33), Private Key = (7, 33) Encrypt M=4: C = 4³ mod 33 = 64 mod 33 = 31 Decrypt C=31: M = 31⁷ mod 33 = 4 ✓
Key Sizes: RSA-1024 (Legacy), RSA-2048 (Standard), RSA-4096 (High Security)
Security Based On: Integer Factorization Problem – Large number factor કરવું computationally impossible

🔐 Diffie-Hellman Key Exchange ★★★★

Insecure channel ઉપર secure key share. "Key Agreement Protocol."

Diffie-Hellman: Public: g (generator), p (prime) Alice: private a → public A = gᵃ mod p Bob: private b → public B = gᵇ mod p Shared Secret: K = Bᵃ mod p = Aᵇ mod p = gᵃᵇ mod p
Security Based On: Discrete Logarithm Problem – gᵃ mod p → a શોધવું impossible!

🔐 ECC (Elliptic Curve Cryptography) ★★★

RSA કરતાં smaller key size + same security. Mobile/IoT ઉપયોગ.

ECC Key Size vs RSA Comparison: RSA 2048-bit ≈ ECC 224-bit (same security!) RSA 3072-bit ≈ ECC 256-bit RSA 15360-bit ≈ ECC 512-bit
ઉપયોગ: Bitcoin (secp256k1 curve), SSL/TLS, Smart Cards, Mobile devices

📝 Digital Signature ★★★★★

Digital Signature = Electronic document ઉપર electronic "signature". Authentication + Non-Repudiation + Integrity.

Digital Signature Process: Signing: Hash(Message) + Private Key → Signature Verification: Signature + Public Key → Hash → Compare with Hash(Message) If Hash matches → Valid Signature ✓ If Hash doesn't match → Invalid/Tampered ✗
PropertyDigital Signature ✓/✗

Non-Repudiation✅ Yes
Confidentiality❌ No (use Encryption for this)
India: IT Act 2000 → Digital Signature legally valid. Certifying Authority (CA) issue કરે.
