CORE CONCEPT · CIA TRIAD
CIA Triad — The Three Pillars of Information Security
CIA ત્રિકોણ — માહિતી સુરક્ષાના ત્રણ સ્તંભ

The CIA Triad is the foundational model for information security. Every security decision is evaluated against these three principles.

C — Confidentiality I — Integrity A — Availability
Principle Gujarati Definition Violated When... MCQ Questions

CONCEPT 1
Confidentiality (ગોપનીયતા) Q1, Q15, Q18, Q23, Q26, Q31, Q34, Q37
Definition: Confidentiality ensures that sensitive or private information is accessible ONLY to those who are authorized to see it. Unauthorized parties are prevented from reading or accessing the data.

How it is achieved:

  • 1Encryption of data (converting readable data to unreadable form)
  • 2Access control mechanisms (passwords, biometrics)
  • 3Data classification (labeling data as public/private/secret)
  • 4Secure communication channels (HTTPS, VPN)
Example

Your online banking password and account data are confidential — only you and the bank can access them. If a hacker reads your bank balance without permission → Confidentiality is VIOLATED.

Confidentiality ≠ Secret. It means the RIGHT people can access it, and the WRONG people cannot.
CONCEPT 2
Integrity (અખંડિતતા) Q2, Q9, Q13, Q20, Q22, Q27, Q30, Q38, Q40
Definition: Integrity ensures that data remains accurate, complete, and UNMODIFIED unless changed by an authorized process. Any unauthorized alteration should be detectable.
Example

You send ₹1000 payment online. During transmission, an attacker changes it to ₹10,000. → Integrity is VIOLATED.

A student's exam marks stored in a database are changed by a hacker → Integrity VIOLATION.

Mechanisms to ensure Integrity:

Hash Functions (MD5, SHA) Digital Signatures Checksums Message Authentication Codes (MAC)
If an UNAUTHORIZED person modifies stored data → PRIMARY violation is INTEGRITY (Q2 ans = D).
CONCEPT 3
Availability (ઉપલબ્ધતા) Q6, Q12, Q17, Q21, Q25, Q28, Q36, Q39, Q42
Definition: Availability ensures that information systems and data are accessible and usable by authorized users WHENEVER they need it. Systems should continue to function reliably even during faults, attacks, or upgrades.
Example

A hospital's patient records system goes down due to a DoS attack. Doctors cannot access patient data during emergency. → Availability VIOLATED.

A system crash preventing access to data = classic Availability violation (Q12 ans = D).

How to maintain Availability:

  • 1Redundant systems (backup servers)
  • 2Failover mechanisms
  • 3Regular system maintenance and updates
  • 4Protection against DoS/DDoS attacks
  • 5Load balancing
CONCEPT 4
Authentication (પ્રમાણીકરણ) Q3, Q7, Q10, Q16, Q35
Definition: Authentication is the process of VERIFYING the identity of a user, device, or system before granting access. It answers "Who are you?"

Three factors of Authentication (3 What's):

FactorTypeExample
Something you KNOWKnowledge FactorPassword, PIN, Security question
Something you HAVEPossession FactorSmart card, OTP token, Phone
Something you AREBiometric FactorFingerprint, Face scan, Iris scan
Example

When you login to Gmail, entering your password verifies your identity → Authentication.
ATM requires card (HAVE) + PIN (KNOW) → Two-factor Authentication (2FA).

Authentication ≠ Authorization. Authentication = WHO you are. Authorization = WHAT you can do.
CONCEPT 5
Non-Repudiation (નોન-રિપ્યુડિએશન) Q8, Q24, Q29, Q33, Q41
Definition: Non-repudiation ensures that a party in a communication CANNOT DENY (repudiate) having sent a message or performed an action. It provides proof of origin and delivery.
NR = Non-Repudiation
Real-world Example

You sign a contract digitally. Later you cannot say "I never signed that!" → Non-repudiation prevents this denial.

In email: Digital signatures provide non-repudiation — the sender cannot later claim they didn't send the email.

How achieved:

Digital Signatures Timestamps Audit Logs Public Key Infrastructure (PKI)
CONCEPT 6
Accountability (જવાબદારી) Q19, Q32, Q43
Definition: Accountability ensures that all actions performed in a system can be TRACED back to a specific user or entity. Every action leaves a trail.
Example

In a company, all file accesses are logged with usernames and timestamps. If data is leaked, the responsible person can be identified → Accountability.

Tools for Accountability:

Audit Logs Access Logs Activity Monitoring User ID Tracking

DEFENSE MECHANISM 1
Firewall (ફાયરવોલ) Q44
Firewall = A network security device (hardware or software) that monitors and controls incoming and outgoing network traffic based on predefined security rules.
FW = FireWall

How it works:

  • 1All network traffic passes through the firewall
  • 2Firewall checks each packet against its rules
  • 3Allowed packets pass through; blocked packets are dropped
  • 4Logs all allowed and blocked traffic
Example

Your home router has a built-in firewall. When you try to access a blocked website, the firewall refuses the connection. Companies use enterprise firewalls to block unauthorized access from the internet.

Packet Filtering Firewall

Examines each packet's source/destination IP and port. Simplest type.

Stateful Inspection Firewall

Tracks the state of connections. More intelligent than packet filtering.

ATTACK 1
Brute Force Attack (બ્રુટ ફોર્સ હુમલો) Q45, Q46, Q52, Q63
Brute Force Attack = An attack that attempts to break a password, encryption key, or login by trying every possible combination systematically until the correct one is found.
Real Example

If a password is 3 digits (000 to 999), a brute force attack tries: 000, 001, 002, 003, ... 999. For a 4-character alphanumeric password = 36⁴ = 1,679,616 combinations to try.

Defense against Brute Force:

Account lockout after N failed attempts CAPTCHA challenges Long complex passwords Multi-factor Authentication (MFA) Rate limiting
Brute Force = "Guess all possible keys until correct one found" (Q45 = B, Q46 = D, Q52 = D, Q63 = C)
DEFENSE MECHANISM 2
Logging System (લોગિંગ સિસ્ટમ) Q47, Q51, Q54, Q60, Q62, Q65
Logging = Recording system events, user activities, access attempts, and security incidents in a log file for later analysis and forensic investigation.

What gets logged:

Login attempts (success & failure) File access events Network connections System errors Security alerts User actions
Example

A bank's server logs show that user "john123" attempted to login 500 times in 1 minute from IP 192.168.1.5 → Brute force attack detected via logs!

SIEM = Security Information and Event Management
SOC = Security Operations Center
DEFENSE MECHANISM 3
Intrusion Detection System — IDS Q48, Q50, Q53, Q58, Q68
IDS = Intrusion Detection System
IPS = Intrusion Prevention System
IDS = A security tool that monitors network and system activities for suspicious behavior, policy violations, or malicious activities, and generates alerts.
IDS (Detection)

MONITORS and DETECTS suspicious activity. Sends alerts but does NOT block. Like a security camera.

IPS (Prevention)

MONITORS, DETECTS and BLOCKS suspicious activity in real-time. Like a security guard + camera.

Example

An IDS detects that a computer is sending 10,000 packets per second (abnormal) and sends an alert to the admin. → Admin can investigate and block the attack.

ATTACK 2
Denial of Service Attack — DoS / DDoS Q49, Q55, Q57, Q69
DoS = Denial of Service
DDoS = Distributed Denial of Service
DoS Attack = An attacker floods a target system with excessive requests/traffic, overwhelming its resources (CPU, memory, bandwidth) so it cannot serve legitimate users.
Example — DoS

Attacker sends 1 million fake requests per second to a shopping website. The server becomes overloaded and crashes. Real customers cannot access the site. → Availability VIOLATED.

Example — DDoS

Instead of one attacker, 10,000 compromised computers (botnet) all attack the same target simultaneously. Much harder to block. This was used against major websites like GitHub (2018).

ATTACK 3
Replay Attack (રિપ્લે હુમલો) Q56, Q61, Q66
Replay Attack = An attacker captures a valid data transmission (like login credentials or authentication token) and retransmits it later to gain unauthorized access, deceiving the system.
Example

Alice sends her login token to a bank server. Attacker captures this token. Later, attacker resends (replays) the same token to the bank → Bank thinks it's Alice and grants access.

Like recording someone's voice saying "Open door" and replaying it to a voice-activated door lock.

Defense against Replay Attacks:

Timestamps (requests expire after a few seconds) Nonces (random one-time numbers) Session tokens (expire after use) Sequence numbers

Eavesdropping (ઈવ્સડ્રૉપિંગ) Q11, Q67
Eavesdropping = Secretly observing/listening to communication WITHOUT altering it. A passive attack that violates Confidentiality.

Key characteristic: PASSIVE — attacker only reads/listens, does NOT modify anything.

Attacker on same Wi-Fi network uses packet sniffer to read all network traffic. They see your unencrypted messages → Eavesdropping.

Traffic Analysis = An attack that studies communication PATTERNS (who communicates with whom, when, how often, message length) rather than the actual message content, to extract useful information.
Example

Even if messages are encrypted, an attacker notices: "This computer sends lots of data to a medical server every day at 9 AM." → Can infer the person visits a doctor regularly, even without reading any message content.

Q64: "Analyzing communication PATTERNS rather than message contents" = Traffic Analysis (Answer = A)

UNIT 03 · Q70–Q95
Modular Arithmetic
મોડ્યુલર અંકગણિત
CORE CONCEPT
What is Modular Arithmetic?
મોડ્યુલર અંકગણિત શું છે?
Modular Arithmetic = A system of arithmetic where numbers "wrap around" after reaching a certain value called the modulus. It deals with remainders.

Think of it like a clock:

Clock Analogy

A 12-hour clock uses modulus 12. After 12, it goes back to 1.
15:00 in 12-hour format = 15 mod 12 = 3 (3 PM)

a mod n = r, where r is the REMAINDER when a is divided by n
Example: 17 mod 5 = 2 (because 17 ÷ 5 = 3 remainder 2)

Why used in Cryptography?

  • 1Modulus keeps results within a fixed range (bounded results)
  • 2Mathematical operations stay manageable in size
  • 3Forms the basis of RSA, Diffie-Hellman, and other public key systems
  • 4One-way property: easy to compute forward, hard to reverse
HOW TO CALCULATE
Step-by-Step Modular Arithmetic Calculation
a mod n = a − (n × ⌊a/n⌋)
⌊a/n⌋ = floor (integer part only, no decimal)

CONCEPT
Congruence in Modular Arithmetic Q72
a ≡ b (mod n) means a and b leave the SAME REMAINDER when divided by n. We say "a is congruent to b modulo n."
a ≡ b (mod n) ⟺ n divides (a - b) evenly
Example: 17 ≡ 2 (mod 5) because both give remainder 2 when divided by 5.
Example

Is 23 ≡ 3 (mod 5)?
23 mod 5 = 3 ✓ and 3 mod 5 = 3 ✓ → YES, they are congruent.

14 ≡ 2 (mod 4)? 14 mod 4 = 2, 2 mod 4 = 2 → YES

Q72 Answer = B: a ≡ b (mod n) means a and b leave the same remainder when divided by n.
PROPERTIES
Properties of Modular Arithmetic Q74, Q82, Q85, Q88, Q91, Q94, Q95

Key Properties used in Cryptography:

PropertyFormulaExample
Addition(a + b) mod n = [(a mod n) + (b mod n)] mod n(7+5) mod 4 = [7 mod 4 + 5 mod 4] mod 4 = [3+1] mod 4 = 0
Subtraction(a - b) mod n = [(a mod n) - (b mod n)] mod n(10-3) mod 4 = [2-3] mod 4 = -1 mod 4 = 3
Multiplication(a × b) mod n = [(a mod n) × (b mod n)] mod n(7×5) mod 4 = [3×1] mod 4 = 3
Why apply modulus after operations? (Q82, Q85, Q88, Q91, Q95)
Modulus is applied to maintain bounded results — keeping numbers within a fixed range. In cryptography, this prevents numbers from growing infinitely large during calculations.
Q74 Answer = C: Distributive property of modulus over addition.

UNIT 04 · Q97–Q115
Symmetric Encryption
સિમેટ્રિક એન્ક્રિપ્શન
CORE CONCEPT
What is Symmetric Encryption?
સિમેટ્રિક એન્ક્રિપ્શન શું છે?
Symmetric Encryption = A type of encryption where the SAME single key is used for BOTH encryption (converting plaintext to ciphertext) and decryption (converting ciphertext back to plaintext). Both sender and receiver must share this secret key securely.
Encryption: Plaintext + Secret Key → Ciphertext
Decryption: Ciphertext + Same Secret Key → Plaintext
Simple Example

Alice and Bob agree on a shared key: "KEY123".
Alice encrypts "HELLO" using KEY123 → sends encrypted gibberish to Bob.
Bob decrypts using the SAME KEY123 → reads "HELLO".

If Eve intercepts, she cannot read it (no key). But if Eve somehow gets KEY123, she can read everything!

PROS & CONS
Symmetric Encryption — Advantages & Disadvantages Q97-Q115
✅ Advantages
  • Fast — much faster than asymmetric encryption (Q98, Q100, Q108, Q109, Q112)
  • Efficient for large data volumes (Q100, Q108, Q109, Q111, Q112, Q113)
  • Simple algorithm design
  • Low computational overhead
❌ Disadvantages
  • Key distribution problem — how to securely share the key? (Q102, Q107, Q110)
  • If key is compromised, all communications are exposed
  • Scalability — N users need N×(N-1)/2 keys
  • No non-repudiation (both parties have same key)
Key Distribution Problem (Q102, Q107, Q110):
Before two parties can use symmetric encryption, they must secretly agree on a shared key. But how? If they meet in person = inconvenient. If they send key over internet = key can be intercepted! This is the main challenge of symmetric encryption.
Q97 = C (secrecy of shared key) | Q98 = B (faster) | Q99 = B (share secret key securely) | Q100 = A (symmetric faster) | Q101 = B (both encryption & decryption) | Q102 = C (secure key distribution) | Q107 = D (key distribution challenges) | Q110 = B (key distribution)
ADVANCED CONCEPT
Hybrid Encryption (Symmetric + Asymmetric) Q104, Q106
Hybrid Encryption = Uses BOTH symmetric and asymmetric encryption together. Asymmetric encryption securely exchanges the symmetric key, then symmetric encryption handles the actual data (for speed).
  • 1Generate a random symmetric session key
  • 2Encrypt the symmetric key using receiver's PUBLIC key (asymmetric)
  • 3Send encrypted symmetric key to receiver
  • 4Receiver decrypts it with their PRIVATE key → gets symmetric session key
  • 5Both parties now use symmetric key to encrypt large amounts of data (fast!)
Real-world Use

HTTPS (SSL/TLS) uses hybrid encryption: RSA (asymmetric) to exchange AES key, then AES (symmetric) to encrypt all web traffic.

Q104 = C: Combination of symmetric and asymmetric for both confidentiality and key exchange. Q106 = C same.
TYPES OF SYMMETRIC CIPHERS
Block Cipher vs Stream Cipher
Block Cipher

Encrypts fixed-size blocks of data (e.g., 128 bits at a time).

Examples: AES, DES, 3DES, Blowfish

Better for stored data and files.

Stream Cipher

Encrypts one bit or byte at a time, continuously.

Examples: RC4, ChaCha20

Better for real-time communication (voice, video).

UNIT 05 · Q116–Q140
Asymmetric Encryption & PKI
અસિમેટ્રિક એન્ક્રિપ્શન અને PKI
CORE CONCEPT
What is Asymmetric Encryption?
અસિમેટ્રિક એન્ક્રિપ્શન શું છે?
Asymmetric Encryption (Public Key Cryptography) = Uses TWO mathematically related but DIFFERENT keys: a Public Key (shared openly) and a Private Key (kept secret). Data encrypted with one key can ONLY be decrypted with the other key in the pair.
For Confidentiality: Encrypt with Receiver's PUBLIC KEY → Decrypt with Receiver's PRIVATE KEY
For Digital Signature: Sign with Sender's PRIVATE KEY → Verify with Sender's PUBLIC KEY
PKI = Public Key Infrastructure
RSA = Rivest–Shamir–Adleman
ECC = Elliptic Curve Cryptography

Example — Confidentiality (Sending Secret Message)

Bob wants to send a secret message to Alice:
1. Bob gets Alice's PUBLIC KEY (available to anyone)
2. Bob encrypts message using Alice's PUBLIC KEY
3. Sends encrypted message to Alice
4. Only Alice can decrypt it using her PRIVATE KEY
→ Even if intercepted, nobody else can read it!

Example — Digital Signature (Authentication)

Alice wants to sign a document:
1. Alice signs (encrypts hash) using her PRIVATE KEY
2. Sends signed document to Bob
3. Bob verifies signature using Alice's PUBLIC KEY
4. If valid → document truly came from Alice (non-repudiation!)

RULE: Encrypt with PUBLIC → Decrypt with PRIVATE. Sign with PRIVATE → Verify with PUBLIC.

MATHEMATICS
Large Prime Numbers in Asymmetric Cryptography Q120, Q125, Q133
Why Prime Numbers? Asymmetric algorithms like RSA rely on the mathematical difficulty of factoring very large numbers that are products of two large prime numbers. This is called the Integer Factorization Problem.
RSA Example

RSA picks two large primes p and q (each hundreds of digits long).
n = p × q (public modulus)

Easy: Given p and q, compute n = p × q in milliseconds.
Hard: Given only n, find p and q → takes billions of years with today's computers!

This one-way difficulty forms the security of RSA.

Q120 = D, Q125 = B, Q133 = B: Large prime numbers are fundamental to asymmetric algorithms.
CONCEPT
Digital Signatures (ડિજિટલ સિગ્નેચર) Q119, Q135, Q138
DS = Digital Signature
Digital Signature = A cryptographic mechanism that provides AUTHENTICATION, INTEGRITY, and NON-REPUDIATION for digital documents and messages. Created using the sender's private key.
  • 1Sender creates a HASH of the message (e.g., SHA-256)
  • 2Sender encrypts the hash using their PRIVATE KEY → this is the digital signature
  • 3Sender sends: original message + digital signature
  • 4Receiver verifies using sender's PUBLIC KEY → decrypts signature to get hash
  • 5Receiver also hashes the received message and compares → if equal, message is authentic!
Real World

Software downloads often include a digital signature. You can verify the downloaded file is really from the original developer (not tampered) by checking the signature with their public key.

Q119 = C: Verify digital signature using sender's PUBLIC KEY.
Q138 = B: Digital signature ensures both authenticity and integrity.
Q135 = C: Digital signature verification confirms authenticity.
CONCEPT
Hash Functions (હેશ ફંક્શન) Q126
Hash Function = A mathematical function that takes variable-length input and produces a FIXED-length output (called hash/digest). It is a one-way function (cannot be reversed).
MD5 = Message Digest 5 (128-bit output) SHA-1 = Secure Hash Algorithm 1 (160-bit) SHA-256 = 256-bit output SHA-3 = Latest SHA standard
Example

SHA-256("HELLO") = "185f8db32..." (64 hex chars = 256 bits, always!)
SHA-256("HELLO!") = "3f79bb7b..." (completely different despite tiny change)

This is called the Avalanche Effect — small input change = completely different hash.

Q126 = C: Hash function produces fixed-length output from variable-length input.
CONCEPT
Key Space (કી સ્પેસ) Q134, Q139
Key Space = The total number of possible keys that can be used in a cipher or encryption system. A larger key space means stronger security (harder to brute force).
Key Space = 2^(key length in bits)
56-bit DES: 2^56 = ~72 quadrillion keys
128-bit AES: 2^128 = impossible to brute force with current tech
Q134 = D, Q139 = C: Key space = total possible keys usable in a cipher.

Access Control (ઍક્સેસ કંટ્રોલ) Q142, Q143, Q147, Q149, Q151, Q153, Q155, Q157, Q160, Q163
Access Control = A security mechanism that restricts users to only the resources and operations they are PERMITTED to access, based on their identity and assigned privileges/roles.

Types of Access Control:

TypeFull FormHow it worksExample
DACDiscretionary ACOwner decides who can access their resourcesFile owner sets read/write permissions

MECHANISM 2
Process Isolation (પ્રોસેસ આઈસોલેશન) Q144, Q154, Q164
Process Isolation = An OS mechanism that keeps processes (running programs) separated from each other in memory, preventing one process from accessing or interfering with another's data or code.

How it works:

  • 1Each process gets its own isolated memory space (Virtual Memory)
  • 2Process A cannot read/write Process B's memory
  • 3If Process A crashes, Process B continues running
  • 4Malware in one process cannot easily spread to others
Example

Your web browser (Chrome) and your music player (Spotify) run as separate isolated processes. A bug in Spotify cannot crash your browser tabs or access your browser passwords → Process Isolation working!

Q144 = A, Q154 = B, Q164 = B: Process isolation = OS mechanism to isolate apps and prevent interference.
MECHANISM 3
Application Control Policy (એપ્લિકેશન કંટ્રોલ) Q145, Q146, Q148, Q152, Q156, Q159, Q162
Application Control Policy / Application Whitelisting = An OS feature that restricts which programs can be executed or installed on a system, preventing unauthorized software from running.
Whitelisting

Only approved applications are allowed to run. Everything else is blocked by default. More secure but less flexible.

Blacklisting

Known bad applications are blocked. Everything else is allowed. Less secure (new malware not yet on blacklist can run).

Example

A company's IT policy says only Microsoft Office, Chrome, and approved business software can run on work computers. An employee cannot install games or unauthorized software. → Application Whitelisting.

Antivirus uses blacklisting — it blocks known viruses/malware.

Q145 = D, Q146 = D, Q148 = A (permission management), Q152 = A, Q156 = A, Q159 = A, Q162 = A: Application control policy prevents unauthorized application execution.
PRINCIPLE
Least Privilege Principle (લઘુત્તમ વિશેષાધિકાર) Q150, Q158, Q161
Principle of Least Privilege (PoLP) = Every user, program, or process should have ONLY the minimum access rights needed to perform its specific job function. No more, no less.
Examples

✅ A data entry clerk should only have READ/WRITE to the data entry form, NOT admin access to the whole database.

✅ A program that only needs to read files should NOT have write permissions.

❌ BAD PRACTICE: Giving everyone admin/administrator rights "just in case".

If an account with least privilege is compromised, the attacker can only do limited damage — they don't have access to the whole system!

Why important?

  • 1Limits damage if account is compromised (Q150)
  • 2Reduces attack surface
  • 3Prevents privilege escalation attacks
  • 4Easier to track what each user does (accountability)
Q150 = B, Q158 = C, Q161 = A: Least privilege principle = restricts users to minimum required access rights.
BEST PRACTICE
Security Updates & Patch Management Q141
Security Update / Patch = Software fix released by vendors to correct vulnerabilities, bugs, or security weaknesses in an operating system or application.
Example

Microsoft releases "Patch Tuesday" security updates every month. If your system is not patched, attackers can exploit known vulnerabilities (like EternalBlue used in WannaCry ransomware).

Always apply security updates promptly → Best OS security practice (Q141 = D)
